Blog posts live at `/home/ng/prod/miozu.com/src/lib/data/blogposts/*.md`. Cross-posts go in the `cross-posts/` subdirectory so they are NOT picked up by the glob pattern `blogposts/*.md`.

The blog API at `src/routes/blog/api/posts.json/+server.js` uses `import.meta.glob('/src/lib/data/blogposts/*.md')` — only top-level `.md` files become published posts.

Frontmatter is parsed by a custom `parseFrontmatter()` function (not gray-matter). Tags must be on a single line: `tags: ["tag1", "tag2"]`. `published: true` is required or the post is filtered out.

The slug route at `src/routes/blog/[slug]/` renders posts via mdsvex with syntax highlighting. JSON-LD schema is generated by the `JsonLd` component — no need to include it in the markdown body.

**Why:** Confirmed by reading the actual +server.js files during the gnosis-mcp launch post.
**How to apply:** Always write to `src/lib/data/blogposts/{slug}.md`. Cross-posts go in `cross-posts/`. JSON-LD goes in the Svelte component, not the markdown.
